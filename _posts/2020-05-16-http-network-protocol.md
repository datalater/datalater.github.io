# 2W1H

## Q1. HTTP는 stateless하다는 게 무슨 말인가?

HTTP 프로토콜은 리퀘스트와 리스폰스를 교환하는 동안에 상태를 관리하지 않는다. 즉, HTTP 프로토콜 레벨에서는 이전에 보냈던 리퀘스트나 이미 되돌려준 리스폰스에 대해서는 전혀 기억하지 않는다.

## Q2. 쿠키는 어떤 맥락에서 왜 만들어졌나?

웹이 진화하면서 스테이트리스 특성만으로는 처리하기 어려운 일이 증가하게 되었다. 예를 들어, 쇼핑 사이트에 로그인했을 때 다른 페이지를 이동하더라도 로그인 상태를 유지할 필요가 있다. 이를 위해서는 누가 어떤 리퀘스트를 보냈는지 파악하기 위해 상태를 유지할 필요가 있다.

HTTP 프로토콜은 상태를 유지하지 않기 때문에, 상태를 유지하고 싶은 요구에 부응하기 위해 쿠키(Cookie)라는 기술이 도입되었다. 쿠키가 있으면 HTTP를 이용한 통신에서도 상태를 계속 관리할 수 있다.

## Q4. 쿠키란 무엇인가?

스테이트리스 프로토콜 HTTP의 특징은 그대로 남겨두고, 클라이언트의 상태를 파악하기 위해 만들어졌다. 리퀘스트와 리스폰스에 쿠키 정보를 추가해서 클라이언트의 상태를 파악한다.

## Q3. 쿠키가 처음 만들어지고 전달되는 전체 흐름을 설명하라.

1. 서버 to 클라이언트:
서버가 쿠키를 발행해서 어느 클라이언트에게 무엇을 전달했는지 기억한다. 서버는 클라이언트에게 리스폰스를 보낼 때 Set-Cookie라는 헤더 필드에 의해 쿠키를 전달한다.
2. 클라이언트 to 서버:
클라이언트는 서버로부터 받은 리스폰스에 있는 쿠키 헤더 필드를 통해 쿠키를 보존한다. 다음 번에 클라이언트가 같은 서버로 리퀘스트를 보낼 때, 자동으로 해당 쿠키 값을 넣어서 송신한다.
3. 서버:
서버는 클라이언트가 보내온 쿠키를 확인해서 어느 클라이언트가 접속했는지 체크하고 서버 상의 기록을 확인해서 이전 상태를 알 수 있다.
