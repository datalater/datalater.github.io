# 01 데이터 타입

## 두 가지 데이터 타입의 차이점

자바스크립트 데이터 타입은 크게 2가지로 분류된다:

* **Primitive Type (기본형)**: 값을 그대로 할당한다.
	* Number
	* String
	* Boolean
	* null
	* undefined
* **Reference Type (참조형)**: 값이 저장된 주소값을 할당한다. (참조)
	* Object
		* Array
		* Function
		* RegExp

기본형과 참조형의 가장 큰 차이점은 값을 그대로 할당하는 것과 값이 저장된 주소값을 할당한다는 것이다. 이러한 차이점이 어떤 의미를 지니는지 알려면 메모리에서 어떤 일이 일어나는지 알아야 한다.

## 메모리에서 일어나는 일

### 기본형 데이터

메모리를 머릿속으로 그리기 위해 다음과 같이 추상화시켜서 상상한다.

| 변수명 | a    | -   | -   |
| ---    | ---  | --- | --- |
| 주소   | @313 | -   | -   |

| 주소   | ... | 313 | 314 | 315 |
| ---    | --- | --- | --- | --- |
| 데이터 | ...   | 10  | -   | -   |

위 두 가지가 별개의 공간을 가지는 것은 아니고 편의상 위처럼 표기한 것이다. 변수명을 담는 공간이 313번보다 앞쪽에 있다고 생각하면 된다.

```javascript
var a;	// (1) 메모리 안에 데이터를 저장할 공간(ex. 313번)을 확보한다. (313번 공간의 데이터는 초기화된 상태)
		// (2) 확보한 공간의 주소값(@313)을 변수명 a와 매칭시킨다.
a = 10;	// (3) 변수 a에 매칭되어 있는 주소값 @313을 찾고 그 주소로 이동해서 그 공간에 10을 저장한다.
```

한 번 더 연습해보자.

```javascript
var b = 'abc';
b = false;
```

메모리에는 다음과 같은 일이 일어난다:

* 데이터를 저장할 공간 314번을 확보한다.
* 확보한 공간의 주소값 @314를 변수명 b와 매칭시킨다.
* 변수명 b와 매칭되어 있는 주소값 @314번으로 이동해서 그 공간에 'abc'를 저장한다.
* 변수명 c와 매칭되어 있는 주소값 @314번으로 이동해서 그 공간에 false를 저장한다.

| 변수명 | a    | b    | -     |
| ---    | ---  | ---  | ---   |
| 주소   | @313 | @314 | -     |

| 주소   | ...  | 313  | 314   | 315 |
| ---    | ---  | ---  | ---   | --- |
| 데이터 | ...    | 10   | false | -   |

즉, 다음과 같이 요약할 수 있다:

* **선언**: 메모리에 공간을 확보하고, 해당 공간의 주소를 변수명과 매칭시키는 과정
* **할당**: 해당 변수가 가리키는 주소의 공간에 값을 저장하는 과정

```javascript
var c = b;
```

* 데이터를 저장한 공간 315번을 확보한다.
* 확보한 공간의 주소값 @315를 변수명 c와 매칭시킨다.
* 변수명 c와 매칭되어 있는 주소값 @315번으로 이동해서 그 공간에 
* (변수명 b와 매칭되어 있는 주소값 @314번으로 이동해서) false를 할당한다.

| 변수명 | a    | b    | c     |
| ---    | ---  | ---  | ---   |
| 주소   | @313 | @314 | @315  |

| 주소   | ...  | 313  | 314   | 315   |
| ---    | ---  | ---  | ---   | ---   |
| 데이터 | ...    | 10   | false | false |

이런 식으로 값을 그대로 할당하는 기본형 데이터는 그 자체로 비교가 가능하다.

```javascript
b === c; //=> true
```

### 참조형 데이터

반면 참조형 데이터는 기본형 데이터와 동작이 다르다. 가장 대표적인 객체를 살펴보자.

```javascript
var obj = {
	a: 1,
	b: 'b'
};
```

선언 과정은 똑같다. 미리 공간을 확보하고 그 공간의 주소를 변수 `obj`와 매칭시키는 것은 기본형 데이터와 동작이 같다.

| 변수명 | obj  | -   | -   |
| ---    | ---  | --- | --- |
| 주소   | @413 | -   | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | -   | -   | ... | -  | -  | -  |

이제 할당 과정을 밟을 차례다. 공간에 값을 할당하려고 보니, 그 값이 기본형이 아니라 참조형이다. 참조형 데이터는 프로퍼티(property)와 값(value) 쌍으로 이루어져있다. 프로퍼티명(property name)은 변수와 비슷한 성질을 지닌다. 변수명과 실제 데이터는 주소값을 통해 연결되어 있다. 프로퍼티와 값 사이에서도 같은 동작을 한다.

각 프로퍼티명들과 해당 프로퍼티의 값을 저장할 주소를 매칭하기 위해서 공간을 새로 확보한다. 여기서는 임의로 1011번을 확보한다.

| 변수명 | obj  | -   | -   |
| ---    | ---  | --- | --- |
| 주소   | @413 | -   | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | -   | -   | ... | { a: @1012, b: @1013 }  | -  | -  |

이제 다시 각각의 주소값마다 기본형 데이터를 할당한다.

| 변수명 | obj  | -   | -   |
| ---    | ---  | --- | --- |
| 주소   | @413 | -   | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | -   | -   | ... | { a: @1012, b: @1013 }  | 1  | 'b'  |

이제 객체의 값을 갖고 있는 주소 @1011번을 @413번에 매칭시킨다.


| 변수명 | obj  | -   | -   |
| ---    | ---  | --- | --- |
| 주소   | @413 | -   | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | @1011 | -   | ... | { a: @1012, b: @1013 }  | 1  | 'b'  |

이런 과정을 데이터 공간에 기본형 데이터가 담길 때까지 반복한다. 즉 참조형 데이터는 결과적으로 기본형 데이터의 집합이라고 볼 수 있다.

```javascript
var obj2 = obj;
```

위 코드는 메모리에서 어떻게 적용될까?

| 변수명 | obj  | obj2 | -   |
| ---    | ---  | ---  | --- |
| 주소   | @413 | @414 | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | @1011 | @1011   | ... | { a: @1012, b: @1013 }  | 1  | 'b'  |

* 데이터를 저장할 공간 414번을 확보한다.
* 확보한 공간의 주소값 @414를 변수명 obj2와 매칭시킨다.
* 변수명 obj2와 매칭되어 있는 주소값 @414번으로 이동해서 그 공간에
* (변수명 obj와 매칭되어 있는 주소값 @413번으로 이동해서) @1011이라는 주소값을 할당한다.

이게 바로 참조가 이루어지는 방식이다. 객체는 @1011번에 저장되어 있는데 그 객체가 저장된 주소만을 복사해온다.

```javascript
obj2.a = 10;
```

* obj2와 매칭되어 있는 @414번 주소로 이동한다.
* @414번 주소의 데이터가 주소값이므로 다시 @1011번 주소로 이동한다.
* @1011번 주소의 데이터에서 프로퍼티 a를 찾는다.
* a와 매칭되어 있는 @1012번 주소로 이동한다.
* @1012번 주소의 데이터에 10을 저장한다.

| 변수명 | obj  | obj2 | -   |
| ---    | ---  | ---  | --- |
| 주소   | @413 | @414 | -   |

| 주소   | ...  | 413 | 414 | ... | 1011 | 1012 | 1013 |
| ---    | ---  | --- | --- | --- | ---  | ---  | ---  |
| 데이터 | ...  | @1011 | @1011   | ... | { a: @1012, b: @1013 }  | 10  | 'b'  |

```javascript
console.log(obj2.a); //=> 10
console.log(obj.a);  //=> 10
obj === obj2;        //=> true
```
obj2가 새로운 객체를 만든 것이 아니라 obj가 바라보던 객체를 함께 바라보고 있기 때문이다.
