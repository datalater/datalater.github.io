# 2W1H

## Q1-1. 프록시 서버란 무엇인가?

클라이언트와 서버 사이에서 리퀘스트와 리스폰스를 중계하는 서버를 말한다.

## Q1-2. 프록시 서버는 왜 사용하는가?

- 캐시를 사용하기 위해서
- 조직 내에 특정 웹 사이트에 접근하지 못하도록 액세스 제한을 하려고
- 액세스 로그를 획득하는 정책을 철저하게 지키려고

## Q1-3. 프록시 사용 방법은 어떻게 구분하는가?

2개의 기준으로 분류한다.

- 캐시하는지 안 하는지
- 메시지를 변경하는지 안 하는지

## Q2-1. 게이트웨이는 프록시와 어떻게 다른가?

게이트웨이나 프록시 모두 클라이언트와 서버를 중계한다. 하지만 게이트웨이는 프록시와 달리 클라이언트와 HTTP 이외의 통신을 하는 서버를 연결한다.

## Q2-2. 게이트웨이를 사용하는 예시는 무엇인가?

HTTP 프로토콜을 이용하지 않는 서버를 연결하는 예시로:

- 데이터베이스에 접속해 SQL 쿼리를 사용해서 데이터를 얻는다거나
- 쇼핑 사이트 등에서 신용카드 결제 시스템과 연계할 때 사용된다.

## Q3. 터널은 언제 사용하는가?

터널은 다른 서버와의 통신 경로를 확립할 때 사용한다. 예를 들어, SSL 같은 암호화 통신을 통해 서버와 안전하게 통신하기 위해 사용된다.

## Q4. 캐시란 무엇인가?

프록시 서버나 클라이언트의 로컬 디스크에 보관된 리소스 사본을 말한다.

## Q5. htm 파일과 html 파일의 차이점은 무엇인가?

기본적으로 같은 역할을 하지만, htm 파일은 html 파일보다 더 오래된 확장자이며, DOS 같은 옛날 OS에서 사용된 확장자이다. 요즘 모던 웹 페이지는 모두 html 파일을 사용한다고 봐도 무방하다.
