# 2W1H

## Q1. HTTP 메시지와 엔티티의 차이점은 무엇인가?

**요약:** 

기본적으로 같으나 HTTP 메시지에 전송 코딩이 적용되면 엔티티 바디의 내용이 달라진다.

**설명:** 

HTTP 메시지는 리퀘스트나 리스폰스에 관한 엔티티 바디를 운반하는 일을 한다.

엔티티는 리퀘스트나 리스폰스의 페이로드(payload, 부가물)로 전송되는 정보를 말한다. 엔티티 헤더 필드와 엔티티 바디로 구성된다.

기본적으로 메시지 바디와 엔티티 바디는 같지만 전송 코딩이 적용된 경우에는 엔티티 바디의 내용이 변화하기 때문에 메시지 바디와 달라진다.

전송 코딩이란 HTTP 데이터를 전송할 때 인코딩(변환)을 실시하는 것을 말한다. 전송 코딩을 하면 다량의 액세스를 효율 좋게 처리할 수 있어서 전송 효율을 높일 수 있다.

## Q2-1. 콘텐츠 코딩이란 무엇인가?

엔티티에 적용하는 인코딩을 가리키는 말로, 엔티티 정보를 유지한 채 용량을 줄이기 위해 압축하는 것을 말한다.

## Q2-2. 주요 콘텐츠 압축의 종류는 무엇인가?

- gzip(GNU zip)
- compress(UNIX 표준 압축)
- deflate(zlib)
- identity(인코딩 없음)

## Q3. 멀티파트란 무엇인가?

멀티파트란 하나의 리퀘스트나 리스폰스로 여러 다른 종류의 데이터를 수용하는 방법을 말한다. 텍스트나 영상, 이미지 같은 서로 다른 여러 가지 데이터를 다룬다.

## Q4. 콘텐츠 네고시에이션이란 무엇인가?

클라이언트와 서버가 제공하는 리소스의 내용에 대해서 교섭하는 것이다. 클라이언트에게 더욱 적합한 리소스를 제공하는 것이 목적이다. 예를 들어, 클라이언트가 사용하는 언어에 따라 영어 버전 리소스나 한국어 버전 리소스를 보낼 수 있다.
