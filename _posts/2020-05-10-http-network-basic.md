# 2W1H

## Q1. "웹"이란 무엇인가?

1. 인터넷으로 연결되어 URL로 웹 문서를 볼 수 있는 정보 시스템을 말한다. (위키피디아)
2. 웹 서버에 웹 페이지가 저장되어 있고 인터넷을 통해 로컬 컴퓨터와 연결되어 있는 일련의 시스템을 웹이라고 한다. (javatpoint)
3. 클라이언트가 웹 리소스를 요청하고 서버가 웹 리소스를 전달해주는 시스템 (jmc)

## Q2. "HTTP"란 무엇인가?

클라이언트가 웹 서버에 웹 리소스를 의뢰하고 웹 서버가 클라이언트에게 웹 리소스를 전달해주는 일련의 흐름을 HTTP 프로토콜이라고 부른다.

## Q3. "TCP/IP"는 무엇인가?

컴퓨터와 네트워크 기기가 서로 통신하기 위해 사용하는 인터넷과 관련된 프로토콜을 종합해서 TCP/IP 프로토콜이라고 부른다.

## Q4. "TCP/IP"는 어떤 계층으로 구성되어 있는가?

1. 애플리케이션 계층: HTTP 클라이언트(브라우저)
2. 트랜스포트 계층: TCP
3. 네트워크 계층: IP
4. 링크 계층: 하드웨어 네트워크

## Q5. "TCP/IP"는 왜 계층화되어 있는가?

1. 첫째, 변경 메리트가 있기 때문이다. 계층이 없으면 어디선가 사양이 변경되었을 때 전체를 바꿔야 한다. 하지만 계층화되어 있으면 사양이 변경된 해당 계층만 바꾸면 된다. 각 계층은 계층이 연결되어 있는 부분만 결정되 어 있어 각 계층의 내부는 자유롭게 설계할 수 있다.
2. 둘째, 설계 메리트가 있기 때문이다. 각 계층은 자기 자신이 담당하는 부분만 고려하면 된다.

## Q6. 클라이언트와 서버가 소통할 때 "TCP/IP" 각 계층은 어떤 역할을 하는가?

1. 애플리케이션 계층(HTTP):
HTTP 클라이언트가 어느 웹 페이지를 보고 싶다는 HTTP 리퀘스트를 보낸다.
2. 트랜스포트 계층(TCP):
애플리케이션 계층에서 받은 HTTP 리퀘스트 데이터(HTTP 메시지)를 통신하기 쉽게 조각내어 TCP 헤더를 추가한 TCP 세그먼트를 네트워크 계층에 보낸다.
3. 네트워크 계층(IP):
전송하는 데이터 최소 단위인 패킷을 상대방 컴퓨터에게 보낸다. 이때 라우터를 거쳐 상대 컴퓨터에 도달하는 여러 가지 경로 중에서 하나의 길을 결정하는 것이 네트워크 계층의 역할이다.
4. 링크 계층:
네트워크에 접속하는 하드웨어적인 면을 다룬다. Ethernet 헤더를 추가한 네트워크 프레임을 서버 측 링크 계층에 보낸다.
5. 서버측 모든 계층:
각 계층을 거칠 때마다 사용한 헤더를 삭제한다.

## Q7. HTTP와 관계가 깊은 IP, TCP, DNS 세 개의 프로토콜은 무엇인가?

1. IP: 네트워크 층에 해당한다. IP는 IP 주소와 MAC 주소에 의존해서 패킷을 라우터를 통해 상대방에게 전달한다.
2. TCP: 트랜스포트 층에 해당한다. 큰 데이터를 TCP 세그먼트라고 불리는 단위 패킷으로 작게 분해해서 상대방에게 보내고, 정확하게 도착했는지 확인하는 역할을 한다.
3. DNS: 애플리케이션 계층에서 클라이언트가 접속하고 싶어하는 도메인 주소의 IP 주소를 알려준다.

## Q8. DNS, TCP, IP는 HTTP와 어떤 관계가 있는지 흐름으로 설명하라.

1. HTTP & DNS:
    1. 클라이언트가 접속하고 싶어하는 도메인 주소의 IP 주소를 DNS가 알려준다.
2. HTTP:
IP 주소를 전달 받은 클라이언트는 담당 웹 서버에 보낼 HTTP 메시지를 작성한다.
3. HTTP & TCP:
    1. TCP는 통신하기 쉽도록 HTTP 메시지를 패킷으로 분해한다.
4. HTTP & IP:
    1. IP는 상대가 어디에 있는지 찾아 라우터를 중계해 가면서 패킷을 배송한다.
5. HTTP & TCP
    1. 서버측 TCP는 상대방으로부터 패킷을 수신해서 도착한 패킷을 조립한다.
6. HTTP:
    1. 웹 서버에 대한 리퀘스트 내용을 처리한다.
7. 리퀘스트 처리 결과도 마찬가지로 TCP/IP 통신 순서대로 클라이언트에 반환
